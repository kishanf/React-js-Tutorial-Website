React.js Development Enviroment Setup

There are many steps involed in this first part I will be showing you the installation of packages and plugins through the command prompt

First we have to create the root folder, this is where all applicaton files will be stored. We will place it on our desktop so it is easy to locate and route to. Open up your CMD or Terminal depending on which OS you're running and enter the command below, i'm using OSX i'll be running all of this in Terminal which will look different but the paths and directories are the same. For those of you who use windows you would enter - \Users\username\Desktop\mkdir reactdemo Check your desktop as you should now have a folder called reactdemo; now we will create an empty json package enter \Users\username\Desktop\reactdemo >npm init, now you will be prompted with a list of instructions for installing packages.

Now we are going to be installing several Babel packages and plugins some local and some global. start by entering \Users\username\Desktop\reactdemo >npm install -g babel and \Users\username\Desktop\reactdemo >npm install -g babel-cli. Babel is a transpiler for JS, it is known for being able to turn ES6 into code that is able to run in browsers and servers, So it is extremly useful when creating web applications.

Next we are going to be installing webpack which is javascript module bundler, it puts all your assests into a dependency graph. enter the following \Users\username\Desktop\reactdemo >npm install webpack --save and \Users\username\Desktop\reactdemo >npm install webpack-dev-server-save. Now we are going to install React while still using the command promt, enter the following \Users\username\Desktop\reactdemo >npm install react --save and \Users\username\Desktop\reactdemo >npm install react-dom --save. There are now 4 babel plugins which you are to install I will list the first one then install the rest via the use of the picture included below; \Users\username\Desktop\reactdemo >npm babel-core.

You're now going to want to open up the package.json and delete the "TEST: echo" line and replace it with webpack-dev-server --hot". we are deleting this since we have no use for testing the classes right now.
We are using webpack because it is a module bundler which helps to split the application into multiple file; this helps by splitting our code into chunks and loading them on demand. Loading them on demand is useful in the sense that it helps us in reducing the loading time of our application. webpack is an extremely helpful tool to have becasue once you specify a single path it does the rest of the work by going through you application and working out how everything is connected throught the values in your css; this tends to be very helpful in regards to single page applications which are becoming the most popular type of web applications.
